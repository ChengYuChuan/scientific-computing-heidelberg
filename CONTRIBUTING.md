# 🤝 Contributing Guide

Thank you for helping keep this resource accurate and useful!

---

## Types of Contributions

### 1. Fix or update course information
Open an Issue using the **Course Info Update** template, or submit a Pull Request directly editing `courses/course-list.md`.

### 2. Add a Q&A entry
If you've received a useful reply from Dr. Winckler:
1. Add a summary entry to `faq/coordinator-qa.md`
2. Optionally upload the anonymized full email to `emails/`

### 3. Add an anonymized email
Follow the rules in [emails/README.md](emails/README.md). No real names or student IDs allowed.

### 4. Start a discussion
For uncertain or open-ended questions (e.g., "how do people usually handle X?"), use **GitHub Discussions** rather than Issues.

---

## Pull Request Guidelines

- Keep PRs focused: one change per PR
- Write a clear PR title, e.g.: `Update: GPU Computing course number corrected`
- If your change is based on a Winckler email, either quote the relevant part or link to the email file

---

## What Not to Do

- ❌ Do not include personal information (names, student IDs, emails)
- ❌ Do not add courses you haven't verified — mark them as ❓ instead
- ❌ Do not delete others' contributions without opening an Issue first

---

## Maintainers

This repo is maintained by M.Sc. Scientific Computing students. If you'd like to become a maintainer, open an Issue and introduce yourself.
